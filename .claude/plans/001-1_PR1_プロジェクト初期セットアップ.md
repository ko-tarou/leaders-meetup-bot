# 001-1 PR1: プロジェクト初期セットアップ

## 概要

Hono + TypeScript + Cloudflare Workers のプロジェクト骨格を構築する。
`wrangler dev` でローカル起動し、ヘルスチェックエンドポイントが動作する状態をゴールとする。

## 作成するファイル

| ファイル | 役割 |
|---------|------|
| `package.json` | 依存関係定義 |
| `tsconfig.json` | TypeScript設定 |
| `wrangler.toml` | Cloudflare Workers設定（D1, KV, Cron bindingsの雛形含む） |
| `.gitignore` | node_modules, dist, .wrangler等 |
| `.dev.vars.example` | ローカル開発用の環境変数テンプレート |
| `src/index.ts` | Honoメインアプリ。ルーティングの骨格 + Cronハンドラの雛形 |
| `src/routes/slack.ts` | Slack Webhook受信ルート（スタブ） |
| `src/routes/api.ts` | 設定CRUD APIルート（スタブ） |
| `src/types/env.ts` | Cloudflare Bindings の型定義 |

## 主要な設定内容

### package.json 依存関係
- `hono` — Webフレームワーク
- `wrangler` — Cloudflare Workers CLI（devDependencies）
- `typescript` — TypeScript（devDependencies）
- `@cloudflare/workers-types` — Workers型定義（devDependencies）

### wrangler.toml
- `compatibility_date` = 最新
- D1 binding の雛形（コメントアウト、DB作成後に有効化）
- Cron Triggers の雛形（コメントアウト）
- `[site]` 設定はPhase 4で追加するため今は含めない

### tsconfig.json
- `target`: `ES2022`
- `module`: `ES2022`
- `moduleResolution`: `bundler`
- `strict`: `true`
- Workers types を include

## テスト方法

- `npm install` が成功すること
- `npx wrangler dev` でローカル起動できること
- `GET /` でヘルスチェックレスポンスが返ること
- `GET /api/health` で JSON レスポンスが返ること
- `POST /slack/events` でスタブレスポンスが返ること
