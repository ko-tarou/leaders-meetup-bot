# 002 全自動サイクル実装

## 概要

スラッシュコマンド不要の完全自動運用を実現する。
Web管理画面で設定したルールに基づき、毎月自動で:
1. 候補日を生成して投票メッセージを送信
2. 締め切り日に自動集計・結果通知
3. 開催前にリマインド送信

## 必要な変更

### DB変更: auto_schedules テーブル追加

ミーティングごとの自動スケジュール設定:

- id: TEXT PK
- meetingId: TEXT → meetings.id
- candidateRule: TEXT (JSON) — 候補日生成ルール
  例: {"type":"weekday","weekday":6,"weeks":[2,3,4]} = 第2〜4土曜日
- pollStartDay: INTEGER — 毎月何日に投票開始 (1-28)
- pollCloseDay: INTEGER — 毎月何日に投票締切 (1-28)
- reminderDaysBefore: TEXT (JSON) — 開催何日前にリマインド [3, 0]
- reminderTime: TEXT — リマインド時刻 "09:00"
- enabled: INTEGER DEFAULT 1
- createdAt: TEXT

### スケジューラ拡張

Cronジョブ（5分間隔）で以下をチェック:
1. 今日がpollStartDayなら → 候補日を自動生成して投票開始
2. 今日がpollCloseDayなら → オープンな投票を自動締切
3. scheduled_jobsのpendingジョブを実行（既存）

### 候補日生成ロジック

candidateRuleに基づき翌月（or当月）の候補日を計算:
- weekday: 0(日)〜6(土)
- weeks: [2,3,4] = 第2,3,4週

### Web管理画面

MeetingDetailに「自動スケジュール」タブ追加
